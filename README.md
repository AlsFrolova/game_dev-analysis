**Анализ поведения пользователей игрового сервиса** — исследование успешности операций, влияния премиум-статуса и возраста, сегментация по платформам, оценка удержания и выявление проблемных зон.

---
## 🎯 Задача

Провести комплексный анализ данных о клиентах и их действиях в игровом приложении:

- Оценить общий **success rate** (доля успешных операций) и его зависимость от платформы, возраста и премиум-статуса.
- Выявить различия в поведении **премиум** и **обычных** пользователей.
- Проанализировать **активность** пользователей (количество операций, время между действиями).
- Оценить **удержание (retention)** через распределение числа операций на пользователя.
- Провести сегментацию пользователей по возрастным группам.
- Найти **аномалии** в данных и сформулировать бизнес-рекомендации.

---

## 📊 Данные

### Таблица `user_data`

| Колонка     | Описание |
|-------------|----------|
| `client`    | Уникальный идентификатор пользователя |
| `premium`   | Наличие премиум-статуса (`True` / `False`) |
| `age`       | Возраст пользователя |

### Таблица `logs`

| Колонка     | Описание |
|-------------|----------|
| `client`    | ID пользователя (связь с `user_data`) |
| `success`   | Результат операции: `True` – успех, `False` – ошибка |
| `platform`  | Платформа (`computer`, `phone`, `tablet`) |
| `time`      | Время операции в формате Unix |

> **Объединение** – через `client` (left join). В результате получена таблица `merged_game_data` для анализа.

---

## 🔬 Этапы анализа

1. **Предобработка**
   - Загрузка данных, проверка типов, дубликатов и пропусков.
   - Преобразование `time` из Unix в `datetime`.
   - Объединение таблиц, обработка `NaN` в поле `age` / `premium`.

2. **Базовые метрики**
   - Общий success rate, количество уникальных пользователей.
   - Среднее число операций на пользователя, доля премиум.
   - Успех по платформам, соотношение успех/неудача.

3. **Анализ интервалов между операциями**
   - Сортировка логов по клиенту и времени.
   - Расчёт `time_diff` (минуты между последовательными операциями).
   - Сравнение среднего и медианного значений, визуализация выбросов.

4. **Сегментация по возрасту**
   - Разбивка на группы: `<18`, `18-24`, `25-34`, `35-49`, `50+`.
   - Расчёт success rate, доли премиум, предпочтений по платформам для каждой группы.
   - Статистическая проверка различий (t-test).

5. **Анализ удержания (retention)**
   - Подсчёт числа операций на каждого клиента.
   - Вычисление доли пользователей, совершивших ровно одну операцию.

6. **Визуализация**
   - Stacked bar chart успеха/неудачи по платформам.
   - Гистограммы возраста для премиум и обычных.
   - Тепловая карта успеха по часам и платформам.
   - Графики сегментных метрик по возрасту.

| **Самая успешная возрастная группа** | **18–24 года: 77,9%** успеха |

| **Медианное время между операциями** | **28 минут** (типичный интервал) |
